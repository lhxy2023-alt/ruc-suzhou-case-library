# 字段映射说明

本次重构以以下两份文件为准：

- `feishu-bitable-sync/config.json`
- `feishu-bitable-sync/data/rebuilt-offers.json`

其中前台案例主数据只使用 `rebuilt-offers.json` 里的 `25fall` 与 `26fall` 真实 offer 记录。`config.json` 代表更宽的同步映射能力，但如果字段没有实际出现在这两季重建数据里，就不能作为当前前台主字段。

## 5. 旧 MVP 遗留字段处理

下列字段来自旧 demo 的假设性案例结构，本次已经删除或降级，不再作为前台主结构：

| 旧字段 | 处理方式 | 原因 |
| --- | --- | --- |
| `title`（手写故事标题） | 删除，改用 `listTitle` | 旧标题不来自真实 Base |
| `region` | 删除 | 当前真实记录没有可靠地区字段 |
| `category` | 删除 | 旧版是人工归类，不是真实字段 |
| `pathType` | 删除 | 旧版是业务推导路径，不是真实字段 |
| `undergraduateBackgroundTag` | 删除 | 旧版是人工标签 |
| `experienceSummary` / `experienceTags` | 删除 | 旧版是扩写或抽象，不是真实记录 |
| `offerDate` / `offerMonthLabel` | 删除 | 当前两季真实重建数据无该字段 |
| `applicantType` | 删除 | 旧版假设字段 |
| `advisorId` / 导师模块 | 删除 | 与真实 offer 记录无直接对应 |
| `timeline` | 删除 | 当前真实记录无该字段 |
| `offerImages` / `relatedPdf` | 删除 | 当前真实数据不支持 |
| `featured` | 删除 | 旧版运营字段 |
| `searchKeywords` | 删除 | 改为前端派生 `searchText` |
| `shareTitle` / `shareDesc` / `ctaText` | 删除 | 旧版营销文案，不再保留 |

## 6. 本次保留原则

- 保留：真实 offer 记录中稳定存在、且适合前台展示的字段。
- 删除或降级：旧 MVP 的业务故事、路径分类、营销文案、顾问信息、时间线、资料包等结构。
- 仅作为派生字段保留：匿名名、搜索文本、学校简称、合并成绩、详情信息块。
- 空字段不展示：`托福`、`GRE/GMAT`、`案例说明/项目说明` 等字段只在有值时渲染。
