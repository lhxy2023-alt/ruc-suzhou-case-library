# 人大中法升学项目库 MVP

一个轻量、纯静态的升学项目库原型，适合上海乐湖修远教育科技有限公司 / 人大中法场景的第一版演示。

## 这版做了什么

- 做了一个最小可运行的 H5 原型
- 支持两个核心视图：
  - 案例列表页
  - 案例详情页
- 每个案例都展示了最关键的结构化信息：
  - 学生匿名名 / 届别
  - 本科专业
  - GPA / 均分区间
  - 语言成绩（可选）
  - 背景标签
  - 最终去向 / 升学路径
- 数据先放在前端静态数组里，方便后续快速改字段、扩案例、转成公众号 H5 或小程序

## 如何运行

### 方式一：直接双击打开
直接打开 `project-library/index.html` 即可预览。

### 方式二：起一个本地静态服务
如果想更接近真实 H5 体验，可以在工作区根目录运行：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/project-library/
```

## 为什么先做成这样

这版故意保持轻量，原因是：

1. 当前目标不是做复杂后台，而是先把“学生背景 → 最终升学路径”展示清楚
2. 先验证信息结构和页面感觉，比一开始做复杂系统更重要
3. 这套字段结构后续可以很容易迁移到：
   - i乐湖 H5 页面
   - 微信小程序
   - 更完整的案例数据库

## 如何改示例数据

演示数据都写在 `app.js` 顶部的 `cases` 数组里；复制一条案例对象再改内容，就是最快的扩充方式。

最适合先扩的字段有：

- `backgroundTags`：最容易先加，能快速补足学生特征、信任来源、诊断标签
- `routeType` / `finalPath`：最适合先扩路径类型，方便把“考研 / 申请 / 待分流”讲清楚
- `destination`：适合补具体学校或项目去向，让案例更像真实成交结果
- `summary` / `diagnosis` / `recommendation`：适合后续慢慢补充，用来增强顾问视角和转化表达

## 下一步最值得补什么

优先级建议：

1. **字段设计再细化**
   - 增加国家 / 地区 / 项目类别 / 是否接受统考 / 是否考数学 等字段
2. **案例筛选增强**
   - 支持按 GPA 区间、路径类型、专业类别筛选
3. **咨询转化入口**
   - 在详情页接入微信咨询 / 升学诊断
4. **路径库化**
   - 不只展示案例，还要展示“什么样的学生适合什么路径”
5. **后台数据化**
   - 后续可以把案例从静态数组迁移到 JSON / CMS / 表格数据源
