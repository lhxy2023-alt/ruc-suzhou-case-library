### Key Decisions Made
- 公司 AI 员工体系的第一阶段先采用 3-agent 精简结构：总控 Agent、获客运营 Agent、销售转化 Agent，而不是一开始铺开更多岗位。这样更贴合当前业务链条“运营获客 → 销售 → 交付”，也更利于建立清晰边界与稳定协作。
- 飞书侧长期采用“1 个 agent 对应 1 个飞书机器人”的原则，避免一个机器人混合扮演总控、运营、销售多个角色造成上下文污染和职责混乱。
- 在当前阶段，现有已接入 OpenClaw 的飞书机器人先作为总控机器人使用；后续再新增“乐湖增长”“乐湖成交”两个飞书机器人，分别承接获客运营与销售转化。
